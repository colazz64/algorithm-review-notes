## Problem Summary

Given an integer array `nums`, return all unique triplets:

```text
[nums[i], nums[j], nums[k]]
```

such that:

```text
i != j
i != k
j != k
nums[i] + nums[j] + nums[k] == 0
```

The answer cannot contain duplicate triplets.

## Core Idea

This problem is a two-pointer problem after sorting.

Steps:

1. Sort the array.
2. Fix one number `nums[i]`.
3. Use two pointers `left` and `right` to find two other numbers.
4. If the sum is too small, move `left`.
5. If the sum is too large, move `right`.
6. If the sum equals `0`, add the triplet and skip duplicates.

After sorting, the array becomes ordered, so we can decide which pointer to move based on the current sum.

## Java Solution

```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        Arrays.sort(nums);

        List<List<Integer>> result = new ArrayList<>();
        int n = nums.length;

        for (int i = 0; i < n - 2; i++) {
            // Skip duplicate fixed numbers
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }

            // Since the array is sorted, if nums[i] > 0,
            // then all following numbers are also positive.
            if (nums[i] > 0) {
                break;
            }

            int left = i + 1;
            int right = n - 1;

            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];

                if (sum < 0) {
                    left++;
                } else if (sum > 0) {
                    right--;
                } else {
                    result.add(Arrays.asList(nums[i], nums[left], nums[right]));

                    left++;
                    right--;

                    // Skip duplicate left values
                    while (left < right && nums[left] == nums[left - 1]) {
                        left++;
                    }

                    // Skip duplicate right values
                    while (left < right && nums[right] == nums[right + 1]) {
                        right--;
                    }
                }
            }
        }

        return result;
    }
}
```

## Python Solution

```python
class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        result = []
        n = len(nums)

        for i in range(n - 2):
            # Skip duplicate fixed numbers
            if i > 0 and nums[i] == nums[i - 1]:
                continue

            # If nums[i] is already positive, no triplet can sum to 0
            if nums[i] > 0:
                break

            left = i + 1
            right = n - 1

            while left < right:
                total = nums[i] + nums[left] + nums[right]

                if total < 0:
                    left += 1
                elif total > 0:
                    right -= 1
                else:
                    result.append([nums[i], nums[left], nums[right]])

                    left += 1
                    right -= 1

                    while left < right and nums[left] == nums[left - 1]:
                        left += 1

                    while left < right and nums[right] == nums[right + 1]:
                        right -= 1

        return result
```

## Mistakes and Notes

### Mistake 1: Missing braces in Java

A common mistake is writing:

```java
else result.add(Arrays.asList(nums[i], nums[left], nums[right]));
right--;
left++;
```

In Java, without braces, only the first line belongs to `else`.

So `right--` and `left++` will run every time, even when `sum < 0` or `sum > 0`.

That changes the pointer movement logic and can skip valid answers.

Better:

```java
else {
    result.add(Arrays.asList(nums[i], nums[left], nums[right]));
    left++;
    right--;
}
```

### Mistake 2: Not skipping duplicates

Because the answer cannot contain duplicate triplets, we need to skip duplicates in two places.

First, skip duplicate fixed numbers:

```java
if (i > 0 && nums[i] == nums[i - 1]) {
    continue;
}
```

Second, after finding a valid triplet, skip duplicate `left` and `right` values:

```java
while (left < right && nums[left] == nums[left - 1]) {
    left++;
}

while (left < right && nums[right] == nums[right + 1]) {
    right--;
}
```

## Pattern

This problem uses the sorted-array two-pointer pattern.

General idea:

```text
Sort first.
Fix one number.
Use left and right to search for the remaining target.
Move pointers based on whether the current sum is too small or too large.
Skip duplicates.
```

## Review Notes

- Sorting is the key step that makes two pointers possible.
- The fixed number is `nums[i]`.
- The remaining problem becomes a two-sum problem on the sorted subarray.
- Duplicate handling is part of the main logic, not an optional detail.
