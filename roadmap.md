# Roadmap

## Goal

Build a structured algorithm review system for coding interview preparation and long-term CS foundation improvement.

## Learning Direction

The current plan is to strengthen algorithm implementation ability first, while gradually preparing for software engineering and AI-related roles.

Current long-term interests:

- Software engineering
- AI infrastructure
- LLM application / agent development
- Python-based development workflow

## Practice Order

| Step | Topic | Focus |
|---|---|---|
| 0 | Programming Basics | Basic syntax, loops, arrays, strings, simple math |
| 1 | Sliding Window | Fixed-size window, variable-size window |
| 2 | Binary Search | Basic binary search, boundary handling |
| 3 | Data Structures | HashMap, prefix sum, stack, queue, heap |
| 4 | Linked List / Tree / Backtracking | Binary tree DFS and recursion |
| 5 | Grid Graph | DFS / BFS on grids |
| 6 | Backtracking | Search tree and pruning |
| 7 | Dynamic Programming | Basic DP patterns |

## Current Stage

I am currently strengthening Python syntax and common APIs.  
Since I am more familiar with Java, I often solve or understand a problem in Java first, then rewrite it in Python.

## Review Principle

For each problem, I want to record:

1. What the problem is really testing.
2. What mistake I made.
3. What template or pattern can be reused.
4. How Java and Python differ in this problem.
5. What I need to review next time.
